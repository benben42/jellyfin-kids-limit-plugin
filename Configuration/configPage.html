Kids Watch-Time Limit

Enforces cumulative daily watch-time limits per kid. Users not enabled below are treated as unlimited adults. See the parent dashboard for live status and bonus time.

Global time-of-day windows

Morning = 00:00 to the first cut-point, Afternoon = between the two, Evening = last cut-point to 24:00.

Parent API token

Leave blank to disable the REST API entirely. Example: POST /KidsLimit/bonus?user=NAME&minutes=30&token=SECRET

Hard enforcement

Recommended if your client (e.g. Android TV) ignores the server's Stop command. When a kid is over their daily or window limit, the plugin blocks that user at the server so playback stops even on stubborn clients, and restores access automatically when they're back under limit, granted bonus, or at midnight. Turn this off (and Save) before uninstalling the plugin to release any active blocks. The dashboard's Stop now button always applies a hard block (until you press Allow again, grant bonus, or midnight), regardless of this setting — using the mode chosen below.
Block all access flips the kid's native access schedule to "never allowed": every request is rejected, so even a stream that's already playing dies within seconds — but the kid is locked out of Jellyfin entirely (browsing too, and some clients drop to an error/login screen). Block playback only is gentler: it turns off the user's media-playback permission, so nothing new can start (including auto-play of the next episode) and the kid can still browse with a friendly "playback not allowed" error — but a client that ignores the Stop command may finish the episode it is currently direct-playing. Transcoded streams are torn down server-side in both modes.
