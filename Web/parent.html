Over-limit alert (auto-stop watchdog)

If a device ignores the Stop command and playback keeps running this many minutes past the limit, you get a push notification (to the targets below) so you know the automatic stop failed. One alert per sitting.

Notification targets

Used for chore-claim pings and the over-limit alert. ntfy: URL = topic URL · Pushover: token + user key · Gotify: URL + app token · Discord/Slack: webhook URL · Telegram: bot token + chat id · Apprise: notify endpoint · Webhook: gets JSON {"title","message"}.

Time-of-day windows

Morning = midnight → first cut, Afternoon = between, Evening = last cut → midnight. Presets can cap each window separately.

Hard enforcement

Recommended when a client (e.g. Android TV) ignores the Stop command. Access is restored automatically when back under limit, on bonus, or at midnight.
